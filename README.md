# 代码可视化工具 Vode

## 项目简介

本项目是一个面向 Python 和 PyTorch 代码的可视化工具，能够以递归的、可选择性展开的方式展示代码的执行流程、数据流动和运行时状态。

---

## 使用方式

Vode 的设计理念是"零侵入"，用户无需修改原有代码，只需在运行命令前加上 `vode` 即可：

```bash
# 原始运行方式
python model/train.py

# 使用 Vode 可视化（全深度信息采集）
vode python model/train.py
```

这种使用方式类似于 `mpirun`、`torchrun` 等工具，通过命令行前缀启动可视化功能。

### 基本用法（仅作示范，不一定是实际功能）

```bash
# 全深度信息采集（默认）
vode python your_script.py

# 指定输出文件
vode --output model_graph.svg python your_script.py

# 启动交互式 Web 界面
vode --web python your_script.py

# 更多选项（后续扩展）
vode --depth 3 --format png python your_script.py
```

运行后，Vode 会：

1. 自动注入追踪代码
2. 执行原始程序
3. 捕获所有层级的计算图和运行时数据
4. 生成可视化输出（静态图或启动 Web 服务器）

## 核心概念

### 统一的计算模型

本工具将所有计算过程统一建模为：

```
Input (dict) → Operation → Output (dict)
```

- **Input**：一个字典，包含该操作所需的所有输入数据（参数、张量、标量等）
- **Operation**：一个计算单元（可以是函数、模块、算子、基础运算等）
- **Output**：一个字典，包含该操作产生的所有输出数据

### 递归展开机制

每个 Operation 都可以被展开为更细粒度的子图：

```
Operation
    ├─ Input (dict)
    ├─ Sub-Operation-1: Input → Op → Output
    ├─ Sub-Operation-2: Input → Op → Output
    ├─ ...
    └─ Output (dict)
```

展开后，原 Operation 的 Input 被分配到各个 Sub-Operation，各个 Sub-Operation 的 Output 汇聚成原 Operation 的 Output。

### 选择性展开

关键特性：用户可以选择性地展开任意单个 Operation，而不必展开同一层级的所有 Operation。

- 可以只展开感兴趣的某个函数，而保持其他函数折叠
- 可以只展开某个模块的内部实现，而保持其他模块为黑盒
- 可以混合不同粒度的视图，在同一张图中同时看到高层抽象和底层细节

## 核心功能

### 1. 数据流捕获

对于每个 Operation，工具捕获：

- **输入字典**：所有输入数据的名称、类型、形状、值
- **输出字典**：所有输出数据的名称、类型、形状、值
- **数据流连接**：哪些输出连接到哪些后续 Operation 的输入

### 2. 变量信息记录

对于数据流中的每个变量，工具记录：

- **变量名称**：在当前作用域中的名称（同一数据在不同 Operation 中可能有不同名称）
- **形状信息**：张量的维度和大小，或标量的类型
- **数据类型**：dtype 和设备信息（cpu/cuda）
- **运行时值**：实际执行时的数值（完整值或统计信息）

### 3. 可配置的显示选项

用户可以控制显示内容：

- **选择性展开**：点击任意 Operation 展开其内部实现
- **信息开关**：选择显示/隐藏变量名、形状、类型、值
- **过滤功能**：按名称、类型、数据类别筛选显示内容

### 4. 变量类型标注

工具支持区分三种数据类型：

- **Argument**：函数或模块的输入数据
- **Parameter**：神经网络的可学习参数（权重、偏置）
- **Constant**：固定的常量值

用户可以手动标注变量类型，工具在可视化时用不同颜色或标记区分。

### 5. 输出格式

- **静态图像**：SVG、PNG、PDF 格式
- **交互式界面**（可选）：Web 界面支持动态展开/折叠、搜索、过滤

## 功能示例

为了具体说明工具的功能，我们以一个简单的神经网络为例。

### 示例代码

```python
import torch
import torch.nn as nn

class SimpleNet(nn.Module):
    def __init__(self):
        super().__init__()
        self.fc1 = nn.Linear(10, 5)
        self.fc2 = nn.Linear(5, 2)
    
    def forward(self, x):
        h = self.fc1(x)
        h = torch.relu(h)
        y = self.fc2(h)
        return y

model = SimpleNet()
input_data = torch.randn(3, 10)
output = model(input_data)
```

```
        └──────────────────────┘
                  │
                  ↓
        ┌──────────────────────┐
        │  SimpleNet.forward   │
        │  (点击展开查看内部)   │
        └──────────────────────┘
                  │
                  ↓
        ┌──────────────────────┐
        │   output: [3, 2]     │
        └──────────────────────┘
```

此时用户看到的是最高层抽象：输入一个名为 input_data 的 [3, 10] 张量，经过 SimpleNet.forward，输出一个名为 output 的 [3, 2] 张量。箭头清晰地显示了数据的流向。

```
- 输入变量名：x

- 输入变量名：input (对应上层的 x)
```

## 关键特性总结

### 1. 统一的递归模型

所有计算都是 `Input(dict) → Operation → Output(dict)` 的形式，可以无限递归展开。

### 2. 选择性展开

可以只展开某一个 Operation，而不影响其他 Operation 的显示状态。这允许用户在同一视图中混合不同粒度的信息。

### 3. 完整的数据流追踪

捕获每个 Operation 的输入输出，以及它们之间的连接关系。

### 4. 丰富的变量信息

记录变量名（区分不同作用域）、形状、类型、运行时值。

### 5. 灵活的显示控制

用户可以选择显示/隐藏各种信息，按需过滤和筛选。

### 6. 参数类型区分

支持标注和区分 Argument、Parameter、Constant。
