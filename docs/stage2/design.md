# Stage 2 UI / UX 设计文档

## 1. 文档目标

本文档定义 [`vode view`](vode/src/vode/cli.py:1) 的桌面端交互界面、信息布局、可视化视图、交互规则和视觉风格。

Stage 2 的产品定位是：

> 一个面向桌面浏览器的、本地运行的、只读的 [`trace.json`](vode/src/vode/trace/serializer.py:54) 交互式查看器。

因此设计重点不是移动端响应式，而是：

- 大屏浏览效率
- 调试工作流顺畅
- 调用树与数据流图双视角切换
- 值信息读取效率
- 搜索、过滤和导出的一致性

---

## 2. 设计原则

### 2.1 桌面优先

界面只针对桌面浏览器设计，默认目标分辨率为：

- 1440 × 900
- 1680 × 1050
- 1920 × 1080

不保证移动端窄屏可用性。

### 2.2 一屏完成主要任务

用户应在一个页面内完成以下动作：

- 加载 trace 结果
- 切换调用树 / 数据流图
- 搜索函数或值
- 选中节点查看详情
- 导出当前视图

### 2.3 以查看和分析为主

Viewer 是分析工具，不是编辑器，因此所有核心交互围绕：

- 浏览
- 定位
- 高亮
- 检查
- 导出

### 2.4 兼容 Stage 1 JSON 心智模型

界面中的实体命名应尽量与 Stage 1 [`GraphSerializer`](vode/src/vode/trace/serializer.py:25) 输出结构保持一致：

- function call
- variable
- edge
- call tree
- dataflow

避免引入与数据结构割裂的新术语。

---

## 3. 桌面界面布局

## 3.1 总体布局

Stage 2 采用三段式桌面布局：

- 顶部 Header
- 左侧导航与函数列表
- 中央主视图区
- 底部值检查器

推荐布局如下：

```text
+--------------------------------------------------------------+
| Header: Vode Viewer | trace.json | Search | [Export] [Reset] |
+----------------------+---------------------------------------+
| Left Panel           | Main View Area                        |
| width: 250px         | flexible                              |
|                      |                                       |
| [Call Tree]          | +-----------------------------------+ |
| [Dataflow]           | | Toolbar: Zoom Pan Fit Export      | |
|                      | +-----------------------------------+ |
| Function List        | |                                   | |
| - main               | |   Call Tree / Dataflow Canvas     | |
| - compute            | |                                   | |
| - add                | |                                   | |
|                      | |                                   | |
| Stats                | +-----------------------------------+ |
| depth max: 4         |                                       |
+----------------------+---------------------------------------+
| Bottom Panel: Value Inspector height: 200px resizable        |
| Selected: compute x=3 y=4 -> tuple                           |
| Args | Returns | Location | Metadata                         |
+--------------------------------------------------------------+
```

### 3.1.1 区域职责

#### Header

用于放置全局信息和高频操作：

- 产品标题
- 当前 trace 文件名
- 全局搜索入口
- 导出按钮
- 重置视图按钮
- 视图状态提示

#### Left Panel

用于放置导航和快速定位能力：

- 视图切换按钮
- 函数列表
- 统计信息
- 常用过滤入口

#### Main View Area

用于承载核心可视化：

- 调用树视图
- 数据流图视图
- 工具栏
- 空状态 / 加载状态 / 错误状态

#### Bottom Panel

用于查看当前选中节点详情：

- 参数列表
- 返回值详情
- 源码位置
- tensor 信息
- 扩展 metadata

---

## 3.2 布局尺寸建议

### 3.2.1 左侧面板

默认宽度：`250px`

约束：

- 最小宽度：`220px`
- 最大宽度：`360px`
- 支持用户拖拽调节

### 3.2.2 底部面板

默认高度：`200px`

约束：

- 最小高度：`160px`
- 最大高度：`360px`
- 支持用户拖拽调节

### 3.2.3 主视图区

剩余空间自适应填充。

要求：

- 不出现双层主滚动区冲突
- 工具栏固定在主视图区顶部
- 图形区域独立滚动或缩放

## 4. 主要视图

## 4.1 Call Tree View

Call Tree View 为默认主视图。

### 4.1.1 展示目标

展示函数调用层级关系，帮助用户回答：

- 谁调用了谁
- 调用深度有多深
- 当前函数的输入和输出是什么
- 某个节点位于哪条调用路径上

### 4.1.2 节点内容

每个函数节点建议展示：

- `display_name`
- 参数预览摘要
- 返回值预览摘要
- 文件名与行号摘要
- 节点状态图标

节点展示示例：

```text
compute
x=3, y=4
-> (7, 12)
test_cli_example.py:7
```

### 4.1.3 树行为

- 单击：选中节点
- 双击：展开 / 折叠节点
- 悬停：显示 tooltip
- 键盘 Space：展开 / 折叠当前节点
- 支持自动滚动到选中项

### 4.1.4 视觉重点

- 当前选中路径高亮
- 已折叠节点显示折叠标记
- 深层节点使用适度缩进
- 参数与返回值使用次级文本样式，避免喧宾夺主

---

## 4.2 Dataflow View

Dataflow View 是第二主视图，用于展示函数间数据传递关系。

### 4.2.1 展示目标

帮助用户回答：

- 某个返回值流向了哪些后续函数
- 某个函数的输入来自谁
- 哪些函数围绕同一对象建立了依赖关系

### 4.2.2 视图元素

- 节点表示函数调用
- 边表示数据传递
- 高亮节点显示上下游关系
- 支持布局切换或布局刷新

### 4.2.3 主要交互

- 单击：选中节点并同步值检查器
- 悬停：显示快速预览
- 滚轮：缩放
- 拖动画布：平移
- 点击工具栏按钮：重置视图、适配屏幕、导出

### 4.2.4 信息层级

Dataflow View 中不应一次性塞入过多值细节。

建议：

- 图上只展示函数名和少量状态
- 参数和返回值详细信息进入底部值检查器
- 将复杂值从主图移出，避免图面板过密

---

## 5. 交互设计

## 5.1 节点交互

### 5.1.1 点击节点

行为：

- 在主视图中标记为选中
- 在 [`ValueInspector.tsx`](vode/docs/stage2/todo.md) 中显示详细信息
- 在左侧函数列表中同步定位
- 若在数据流图中，还应高亮上下游节点

### 5.1.2 双击节点

行为：

- 在 Call Tree 中展开 / 折叠子节点
- 在 Dataflow View 中可保留为聚焦或适配视图，不强制要求双击行为一致

### 5.1.3 悬停节点

行为：

- 显示 tooltip
- 内容包括函数名、文件名、行号、参数摘要、返回值摘要

tooltip 示例：

```text
compute
qualified: test_cli_example.compute
args: x=3, y=4
return: (7, 12)
location: test_cli_example.py:7
```

### 5.1.4 右键菜单

建议提供以下操作：

- 复制节点信息
- 导出当前子树
- 仅显示相关节点
- 高亮上游
- 高亮下游

右键功能属于增强项，若 Stage 2 初版实现成本偏高，可后置为第二优先级。

---

## 5.2 面板交互

### 5.2.1 左侧面板拖拽

- 用户可调整宽度
- 调整后记住本地偏好设置

### 5.2.2 底部面板拖拽

- 用户可调整高度
- 面板展开时不应遮挡 Header 和主视图区控制栏

### 5.2.3 空状态设计

当无选中节点时，值检查器显示：

```text
未选择节点
请在调用树或数据流图中点击一个函数节点查看详情
```

---

## 5.3 搜索与过滤

### 5.3.1 搜索设计

搜索应支持实时输入，但避免每次按键都对大型图全量重算。

建议行为：

- 输入后延迟触发搜索
- 展示结果数量
- 支持跳转到上一项 / 下一项结果
- 支持搜索函数名、限定名、文件路径、参数值、返回值文本

### 5.3.2 过滤设计

建议提供以下过滤器：

- 调用深度
- 模块前缀
- 返回值类型
- 是否只显示相关子树
- 是否只显示数据流主链路

过滤应遵循：

- 不修改原始数据
- 可一键清空
- 在 Header 或侧边栏显示当前过滤摘要

## 6. 颜色方案

## 6.1 函数节点颜色

- 普通函数：`#1890ff`
- 选中函数：`#fa8c16`
- 错误函数：`#f5222d`

### 6.1.1 颜色语义

- 蓝色表示默认可浏览状态
- 橙色表示当前关注焦点
- 红色表示异常、错误或失败返回相关节点

如果 Stage 1 当前还没有异常节点结构，红色状态可以先作为保留视觉语义。

## 6.2 数据流边颜色

- 正常流：`#52c41a`
- 高亮流：`#faad14`

## 6.3 界面背景颜色

- 背景：`#f0f2f5`
- 面板：`#ffffff`
- 边框建议：浅灰中性线

### 6.3.1 深浅层次建议

- Header 使用略深于页面背景的中性底色
- 左侧面板和底部面板保持白底
- 图形画布区域保持白底或极浅灰底，便于导出

---

## 7. 组件细节

## 7.1 Left Panel

Left Panel 包含以下区域：

### 7.1.1 视图切换按钮

- Call Tree
- Dataflow

要求：

- 当前视图状态清晰
- 切换时保留选中节点上下文

### 7.1.2 函数列表

显示内容：

- 函数显示名
- 调用深度缩进
- 可选图标状态
- 搜索命中高亮

要求：

- 可搜索
- 可滚动
- 可点击定位主视图

### 7.1.3 统计信息

建议显示：

- 总函数数
- 总变量数
- 最大深度
- 调用树根节点数

---

## 7.2 Main View

Main View 分为工具栏与画布区域。

### 7.2.1 工具栏

建议提供：

- 放大
- 缩小
- 适配屏幕
- 重置视图
- 导出
- 布局刷新

### 7.2.2 可视化画布

- Call Tree 推荐 SVG 渲染
- Dataflow View 可由 Cytoscape.js 负责内部渲染
- 画布必须支持明确的焦点与选中反馈

### 7.2.3 缩略图导航

可选能力：

- 当数据流图较大时提供 mini map
- Stage 2 可标注为增强项，不强制首版实现

---

## 7.3 Value Inspector

Value Inspector 是信息密度最高的面板。

建议分为多个区块或标签页：

- Overview
- Args
- Returns
- Location
- Metadata

### 7.3.1 Overview

显示：

- 当前函数名
- 限定名
- 调用深度
- 父节点摘要
- 子节点数量

### 7.3.2 Args

显示：

- 参数名
- 类型名
- 预览值
- tensor 元信息

### 7.3.3 Returns

显示：

- 返回槽位
- 类型名
- 预览值
- tensor 统计信息

### 7.3.4 Location

显示：

- 文件路径
- 行号
- 可选源码片段入口

### 7.3.5 Metadata

显示：

- Stage 1 中的 `metadata`
- 未来扩展字段，例如执行时间、异常信息

---

## 8. 键盘快捷键

建议支持：

- `Ctrl+F`：打开搜索
- `Ctrl+E`：导出当前视图
- `Ctrl+R`：重置当前视图
- `Space`：展开 / 折叠选中节点
- `Arrow Keys`：导航节点

### 8.1 交互规则建议

- 快捷键只在 Viewer 主区域聚焦时生效
- 输入框聚焦时避免拦截文本编辑相关按键
- 导出等高风险动作应有轻提示反馈

---

## 9. 状态与反馈设计

## 9.1 加载状态

加载 trace 时建议显示：

```text
正在加载 trace 数据...
正在构建索引和视图模型
```

## 9.2 空结果状态

搜索无结果时显示：

```text
未找到匹配项
请尝试函数名、文件名或参数关键字
```

## 9.3 错误状态

如果文件结构与 Stage 1 格式不兼容，应显示：

```text
无法加载 trace 文件
原因：缺少 graph.function_calls 或版本不兼容
```

要求：

- 错误信息尽量明确指出字段名
- 避免只显示通用报错

## 10. 典型用户流程

## 10.1 流程一：查看调用树

```text
打开 Viewer
  -> 默认进入 Call Tree
  -> 点击 compute
  -> 底部显示参数和返回值
  -> 双击展开子节点
  -> 导出当前树图
```

## 10.2 流程二：分析数据流

```text
切换到 Dataflow
  -> 点击某个函数节点
  -> 高亮上下游
  -> 在 Value Inspector 中查看相关值
  -> 按类型过滤只看 tensor 相关节点
```

## 10.3 流程三：快速搜索定位

```text
按 Ctrl+F
  -> 输入 forward
  -> 跳转到命中节点
  -> 查看源码位置和返回值摘要
```

## 12. 设计结论

Stage 2 的 UI / UX 设计应围绕以下核心判断展开：

- 默认主入口是 Call Tree View
- Dataflow View 是第二核心视图
- 所有复杂值详情统一沉到底部检查器
- 左侧面板负责导航和统计，主视图负责图形，底部面板负责细节
- 所有布局和交互都以桌面浏览效率为优先目标

该设计与 [`capabilities.md`](vode/docs/stage2/capabilities.md) 的能力范围和 [`todo.md`](vode/docs/stage2/todo.md) 的技术实现计划保持一致，可直接作为后续切换到实现模式时的界面设计基线。
